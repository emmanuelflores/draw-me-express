Theory One

Blocking and Non-Blocking servers

clients request things from servers

Traditionally servers take care of these request by creating a thread for each one of the request, meaning that the server will block the request until it is completed.


But node is different

Node is single threaded making use of an event loop where functions listen to events, acts and sleeps.


We gain control and scalability!
